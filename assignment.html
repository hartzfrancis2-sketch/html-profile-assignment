Question 1: block vs inline elements

Block Elements: this elements take up the full width available, with a new line before and after.

Examples of Block Elements:

  • div
  • paragraph

Inline Elements: these elements only take up as much width as necessary, and do not start on a new line.

Examples of Inline Elements:

  • span
  • strong

Question 2: true or false an inline element can be a block element?

False, an inline element cannot be a block element because they have different display behaviors in HTML.


Question 3: what happens when a div element is place in an html by default?

By default, a div element is displayed as a block-level element, which means it starts on a new line and takes up the full width available.

Answer:

The purpose of the class attribute in HTML is to assign one or more class names
to an element, allowing for easier targeting
and styling of multiple elements with the same class
using CSS and JavaScript.

Bonus1:Explain the difference between using float, flexbox, and grid for laying out elements.
when would you use each one

Answer:

Float: Float is an older layout technique that allows elements to be taken out of the normal document flow and positioned to the left or right.
It is best suited for simple layouts, such as wrapping text around images. However, it can lead to issues with clearing and overlapping content.

flexbox: Designed for one-dimensional layouts, arranging items in a single row or column. Ideal for distributing space and aligning items within a container.
and it is responsive will also require a parent div

Grid: Designed for two-dimensional layouts, arranging items in both rows and columns simultaneously.
Ideal for complex page layouts and defining relationships between different areas of a design.
it also requires a parent div
